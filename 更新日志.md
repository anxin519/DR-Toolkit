## v2.1 (2025-03-13)

### 🎉 新增功能

#### 多节点发送功能
- ✅ 支持配置多个目标节点
- ✅ 可视化节点列表管理
- ✅ 复选框选择发送目标
- ✅ 一键发送到多个节点
- ✅ 批量连接测试
- ✅ 实时发送进度显示
- ✅ 详细的发送结果统计

### 📋 功能详情

#### 1. 节点管理界面

**节点列表显示：**
- 表格显示所有配置的节点
- 列：选中状态、名称、AE Title、Host、Port、状态
- 双击切换选中状态
- 选中节点高亮显示（绿色背景）

**快捷操作：**
- 全选按钮：一键选中所有节点
- 全不选按钮：一键取消所有选择
- 测试选中按钮：批量测试选中节点的连接

**节点编辑：**
- 添加节点：输入信息后点击"添加节点"
- 更新节点：选中节点后修改信息，点击"更新节点"
- 删除节点：选中节点后点击"删除节点"

#### 2. 多节点发送流程

**步骤：**
1. 在节点列表中勾选要发送的目标节点
2. 添加要发送的DICOM文件
3. 点击"发送到选中节点"按钮
4. 确认发送对话框
5. 实时显示发送进度
6. 查看详细发送结果

**发送进度：**
- 显示当前正在发送的节点
- 显示总体进度（已完成/总任务数）
- 百分比进度显示

**发送结果：**
- 按节点分组显示结果
- 每个节点显示成功和失败数量
- 如有错误显示详细错误信息

#### 3. 批量连接测试

**功能：**
- 选中多个节点后点击"测试选中"
- 依次测试每个节点的连接
- 实时更新节点状态列（✓ 或 ✗）
- 测试完成后显示汇总结果

**测试结果显示：**
```
连接测试结果:

PACS主服务器: ✓ 成功

备份服务器: ✗ 失败
  连接失败！
  错误类型: 连接超时
  详细信息: 无法连接到 192.168.1.101:104
  建议: 检查网络连接和防火墙设置
```

### 💡 使用场景

#### 场景1：同时发送到主备PACS
```
1. 勾选"PACS主服务器"和"备份服务器"
2. 添加要发送的文件
3. 点击"发送到选中节点"
4. 文件会同时发送到两个服务器
```

#### 场景2：多院区数据分发
```
1. 配置多个院区的PACS节点
2. 勾选需要接收数据的院区
3. 批量发送影像数据
4. 查看每个院区的接收情况
```

#### 场景3：测试环境验证
```
1. 配置测试、预生产、生产环境节点
2. 先测试连接确保可达
3. 选择要发送的环境
4. 发送测试数据验证
```

### 🔧 技术实现

**节点存储：**
- 配置文件：`config/app_config.json`
- 字段：name, ae, host, port
- 持久化存储，程序重启后保留

**并发发送：**
- 使用后台线程处理
- 顺序发送到各个节点
- 不阻塞UI操作
- 实时进度反馈

**错误处理：**
- 单个节点失败不影响其他节点
- 详细记录每个节点的发送结果
- 错误信息记录到日志文件

### 📊 性能优化

**发送效率：**
- 每个节点独立发送

- 支持大批量文件发送

**内存优化：**
- 文件按需读取
