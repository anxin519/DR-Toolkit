# 多节点发送功能 - 使用指南

## 功能概述

多节点发送功能允许您将DICOM文件同时发送到多个目标PACS服务器，大大提高了数据分发的效率。

## 使用步骤

### 步骤1：配置节点

#### 方法A：添加新节点
1. 在右侧"添加/编辑节点"区域填写：
   ```
   名称: PACS主服务器
   AE:   PACS_SCP
   Host: 192.168.1.100
   Port: 104
   ```
2. 点击"添加节点"按钮
3. 节点会出现在左侧列表中

#### 方法B：使用已有节点
- 程序会自动加载配置文件中的节点
- 首次运行会有一个默认节点

### 步骤2：选择目标节点

#### 方法A：手动选择
- 双击节点行切换选中状态
- ☐ 表示未选中
- ☑ 表示已选中
- 选中的节点会显示绿色背景

#### 方法B：批量选择
- 点击"全选"按钮：选中所有节点
- 点击"全不选"按钮：取消所有选择

### 步骤3：测试连接（可选但推荐）

1. 选中要测试的节点
2. 点击"测试选中"按钮
3. 等待测试完成
4. 查看状态列：
   - ✓ 表示连接成功
   - ✗ 表示连接失败
5. 查看详细测试结果对话框

**测试结果示例：**
```
连接测试结果:

PACS主服务器: ✓ 成功
  响应时间: 45ms
  远程AE: PACS_SCP

备份服务器: ✗ 失败
  错误类型: 连接超时
  详细信息: 无法连接到 192.168.1.101:104
  建议: 检查网络连接和防火墙设置

测试服务器: ✓ 成功
  响应时间: 12ms
  远程AE: TEST_SCP
```

### 步骤4：添加文件

#### 方法A：添加单个文件
1. 点击"添加文件"按钮
2. 选择一个或多个DCM文件
3. 文件路径会显示在列表中

#### 方法B：添加整个文件夹
1. 点击"添加文件夹"按钮
2. 选择包含DCM文件的文件夹
3. 程序会递归扫描所有DCM文件
4. 所有文件路径会显示在列表中

### 步骤5：发送文件

1. 确认已选中目标节点（绿色背景）
2. 确认文件列表正确
3. 点击"发送到选中节点"按钮
4. 在确认对话框中查看信息：
   ```
   将 100 个文件发送到以下节点:
   PACS主服务器, 测试服务器
   
   确定继续吗？
   ```
5. 点击"是"开始发送

### 步骤6：查看进度

发送过程中会显示：
```
正在发送到 PACS主服务器...
进度: 50/200 (25%)
```

- 第一个数字：已完成的任务数
- 第二个数字：总任务数（文件数 × 节点数）
- 百分比：总体进度

### 步骤7：查看结果

发送完成后会显示详细结果：
```
发送完成！

总文件数: 100

【PACS主服务器】
  ✓ 成功: 100

【测试服务器】
  ✓ 成功: 98
  ✗ 失败: 2
```

## 实际应用场景

### 场景1：主备同步

**需求：** 将影像同时发送到主PACS和备份PACS

**操作：**
1. 配置两个节点：
   - PACS主服务器（192.168.1.100）
   - PACS备份服务器（192.168.1.101）
2. 勾选两个节点
3. 添加要发送的文件
4. 点击发送

**优势：**
- 一次操作，两处存储
- 提高数据安全性
- 节省操作时间

### 场景2：多院区分发

**需求：** 将检查结果发送到多个院区

**操作：**
1. 配置多个院区节点：
   - 总院PACS
   - 分院A PACS
   - 分院B PACS
   - 分院C PACS
2. 根据需要勾选目标院区
3. 批量发送影像数据

**优势：**
- 快速数据分发
- 支持选择性发送
- 统一管理多个节点

### 场景3：环境验证

**需求：** 在不同环境测试DICOM数据

**操作：**
1. 配置多个环境节点：
   - 开发环境（127.0.0.1:11112）
   - 测试环境（192.168.1.200）
   - 预生产环境（192.168.1.201）
2. 先测试连接确保可达
3. 选择要验证的环境
4. 发送测试数据

**优势：**
- 快速环境切换
- 批量测试连接
- 避免配置错误

### 场景4：数据备份

**需求：** 定期备份影像到多个存储位置

**操作：**
1. 配置多个备份节点
2. 全选所有备份节点
3. 添加要备份的文件夹
4. 批量发送

**优势：**
- 多重备份保障
- 自动化备份流程
- 减少人工操作

## 高级技巧

### 技巧1：节点命名规范

建议使用清晰的命名规范：
```
生产-PACS主
生产-PACS备
测试-PACS
开发-PACS
院区A-PACS
院区B-PACS
```

### 技巧2：快速测试所有节点

```
1. 点击"全选"
2. 点击"测试选中"
3. 查看哪些节点可用
4. 取消不可用的节点
```

### 技巧3：分批发送

如果文件很多，可以分批发送：
```
1. 第一批：添加部分文件 → 发送
2. 清空列表
3. 第二批：添加剩余文件 → 发送
```

### 技巧4：节点分组

通过命名实现逻辑分组：
```
主要节点：
- PACS主
- PACS备

测试节点：
- 测试环境
- 开发环境

院区节点：
- 院区A
- 院区B
```

### 技巧5：保存常用配置

节点配置会自动保存到配置文件，下次启动自动加载。

## 注意事项

### ⚠️ 重要提示

1. **网络带宽**
   - 同时发送到多个节点会占用更多带宽
   - 建议根据网络情况选择节点数量

2. **发送时间**
   - 发送时间 = 文件数 × 节点数 × 单文件时间
   - 100文件 × 3节点 ≈ 1.5分钟

3. **错误处理**
   - 单个节点失败不影响其他节点
   - 查看详细结果了解失败原因

4. **节点配置**
   - 配置保存在 `config/app_config.json`
   - 可以手动编辑配置文件
   - 建议定期备份配置文件

5. **日志记录**
   - 所有发送操作都会记录到日志
   - 日志位置：`logs/app_YYYY-MM-DD.log`
   - 可用于问题排查

## 故障排除

### 问题1：节点测试失败

**可能原因：**
- 网络不通
- 防火墙阻止
- 目标服务未运行
- AE Title不匹配

**解决方法：**
1. 检查网络连接
2. 检查防火墙设置
3. 确认目标服务运行状态
4. 验证AE Title配置

### 问题2：发送部分失败

**可能原因：**
- 文件损坏
- 网络不稳定
- 目标存储空间不足

**解决方法：**
1. 查看详细错误信息
2. 检查失败的文件
3. 重新发送失败的文件

### 问题3：发送速度慢

**可能原因：**
- 网络带宽限制
- 文件过大
- 节点过多

**解决方法：**
1. 减少同时发送的节点数
2. 分批发送文件
3. 检查网络状况

### 问题4：节点列表为空

**可能原因：**
- 配置文件不存在
- 配置文件格式错误

**解决方法：**
1. 手动添加节点
2. 检查配置文件格式
3. 删除配置文件让程序重新创建
