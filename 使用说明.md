# DICOM运维工具 v2.0 - 使用说明

## 快速开始

### 安装依赖
```bash
pip install -r requirements_full.txt
```

### 运行程序
```bash
python src/main_complete.py
```

## 功能说明

### 1. 📤 发送页面

**功能：** 批量发送DICOM文件到一个或多个远程PACS

**使用步骤：**

#### 方式一：使用已配置的节点
1. 在节点列表中双击勾选要发送的目标节点
   - 可以选择一个或多个节点
   - 选中的节点会显示绿色背景
2. 点击"测试选中"按钮测试连接（可选）
3. 添加文件或文件夹
4. 点击"发送到选中节点"按钮
5. 确认发送对话框
6. 查看发送进度和结果

#### 方式二：添加新节点
1. 在右侧"添加/编辑节点"区域填写信息：
   - 名称：节点的显示名称（如"PACS主服务器"）
   - AE Title：远程节点的AE Title
   - Host：远程节点的IP地址或主机名
   - Port：远程节点的端口号
2. 点击"添加节点"按钮
3. 新节点会出现在左侧列表中
4. 勾选该节点并发送文件

**快捷操作：**
- **全选**：一键选中所有节点
- **全不选**：一键取消所有选择
- **测试选中**：批量测试选中节点的连接状态

**节点管理：**
- **添加节点**：填写信息后点击"添加节点"
- **更新节点**：选中节点，修改信息后点击"更新节点"
- **删除节点**：选中节点后点击"删除节点"

**测试结果说明：**
- ✓ 连接成功：显示响应时间和远程AE
- ✗ 连接超时：检查网络和防火墙
- ✗ AE不匹配：检查AE Title配置
- ✗ 连接被拒绝：检查目标服务是否运行

**发送结果显示：**
```
发送完成！

总文件数: 100

【PACS主服务器】
  ✓ 成功: 100

【备份服务器】
  ✓ 成功: 98
  ✗ 失败: 2
```

**使用场景：**
- 同时发送到主备PACS
- 多院区数据分发
- 测试环境验证
- 数据备份

### 2. 📥 接收页面

**功能：** 启动DICOM SCP服务接收影像

**使用步骤：**
1. 配置本地AE Title和端口
2. 设置存储路径
3. 点击"▶ 启动"按钮
4. 日志区域显示接收记录

**注意事项：**
- 默认端口：11112
- 确保端口未被占用
- 存储路径会自动创建

### 3. ✏️ 编辑器页面

**功能：** 查看和编辑DICOM文件

**左侧 - 图像显示：**
- 自动显示DICOM图像
- 窗宽窗位滑块手动调整
- 肺窗预设按钮（-600/1500）
- 首次打开自动计算最佳窗宽窗位

**右侧 - 标签显示：**
- 显示所有DICOM标签
- 可滚动查看

**操作按钮：**
- **打开**：选择DCM文件
- **匿名化**：患者信息匿名化（保留病历号后4位）
- **修改UID**：原UID + 时间戳
- **计算年龄**：仅当文件无年龄时显示
- **保存**：保存修改后的文件

**计算年龄逻辑：**
- 检测PatientAge标签
- 如果为空或不存在，显示"计算年龄"按钮
- 点击后根据出生日期和检查日期自动计算
- 计算完成后按钮自动隐藏

### 4. 📁 文件浏览器页面

**功能：** 批量处理DICOM文件

**使用步骤：**
1. 点击"选择文件夹"
2. 程序自动递归扫描所有DCM文件
3. 表格显示文件信息：
   - 路径、文件名
   - 患者姓名、病历号
   - 性别、年龄
   - 检查日期、模态

**导出Excel：**
- 点击"导出Excel"按钮
- 选择保存位置
- 自动生成格式化的Excel文件
- 包含表头、自动列宽、冻结首行

**批量操作：**

1. **批量匿名化**
   - 处理所有扫描的文件
   - 保留病历号后4位
   - 前缀：ANON（可配置）
   - 自动保存

2. **批量计算年龄**
   - 仅处理无年龄的文件
   - 根据出生日期和检查日期计算
   - 格式：035Y
   - 自动保存

3. **批量修改UID**
   - 原UID + 时间戳
   - 同一Study保持StudyInstanceUID一致
   - 同一Series保持SeriesInstanceUID一致
   - 每个实例SOPInstanceUID唯一
   - 自动保存

**进度显示：**
- 实时进度条
- 处理完成后显示统计信息

## 配置文件

### 配置文件位置
`config/app_config.json`

### 首次运行
程序会自动创建默认配置文件

### 配置项说明

```json
{
  "remote_nodes": [
    {
      "name": "默认PACS",
      "ae": "PACS_SCP",
      "host": "127.0.0.1",
      "port": 104
    }
  ],
  "local_scp": {
    "ae": "DICOM_TOOL",
    "port": 11112,
    "storage_path": "./storage"
  },
  "ui_settings": {
    "theme": "cosmo",
    "window_presets": {
      "lung": {"center": -600, "width": 1500}
    }
  },
  "anonymize": {
    "keep_last_digits": 4,
    "prefix": "ANON"
  },
  "uid_strategy": {
    "method": "append_timestamp"
  }
}
```

### 可用主题
cosmo, flatly, litera, minty, lumen, sandstone, yeti, pulse, united, morph, journal, darkly, superhero, solar, cyborg, vapor, simplex, cerculean

## 日志文件

### 日志位置
`logs/` 目录

### 日志文件
- `app_YYYY-MM-DD.log`：应用日志
- `forward_YYYY-MM-DD.log`：转发日志
- `scp_YYYY-MM-DD.log`：SCP接收日志

### 日志特性
- 按日期自动分文件
- 单文件最大10MB
- 自动轮转，保留30个备份
- UTF-8编码

## 常见问题

### 1. PyQt5无法使用
**解决方案：** 使用Tkinter版本（已实现）
```bash
python src/main_complete.py
```

### 2. 图像无法显示
**可能原因：**
- 文件不包含像素数据
- 图像格式不支持

**解决方案：**
- 检查文件是否完整
- 查看标签信息确认是否有PixelData

### 3. 连接测试失败
**检查项：**
- 网络连接
- 防火墙设置
- 目标服务是否运行
- AE Title是否正确
- 端口是否正确

### 4. SCP启动失败
**可能原因：**
- 端口被占用
- 权限不足（端口<1024需要管理员权限）

**解决方案：**
- 更换端口（建议使用11112）
- 以管理员身份运行

### 5. 批量处理速度慢
**优化建议：**
- 文件数量较多时耐心等待
- 查看进度条了解进度
- 不要在处理过程中关闭程序
