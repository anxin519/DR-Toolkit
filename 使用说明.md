# DICOM 运维工具 - 使用说明

## 启动方式

双击 `启动工具.bat` 或在项目根目录执行：

```bash
python src/main_complete.py
```

---

## 📤 发送页

### 节点管理
1. 右侧填写节点信息后点"添加节点"
2. 双击节点列表中的行可将参数填入右侧编辑区（用于修改）
3. 点击第一列（✓）勾选/取消勾选节点

### 发送文件
1. 点"添加文件"或"添加文件夹"选择 DCM 文件
2. 勾选目标节点（支持多选）
3. 点"发送到选中节点"

### 测试连接
勾选节点后点"测试选中"，对每个节点发送 C-ECHO，失败时显示具体原因。

---

## 📥 接收页

### 启动接收服务
1. 填写本地 AE Title、端口、存储路径（配置自动保存）
2. 点"▶ 启动"

### 自动转发
勾选"启用自动转发"后，收到文件会自动加入转发队列。

**无转发规则时**：转发到发送页中勾选的所有节点。
**有转发规则时**：按规则匹配，只转发到匹配的节点。

### 配置转发规则
点"➕ 添加规则"，填写：
- 模态过滤：如 DR、CT，空则匹配所有模态
- 来源AE过滤：如 DR_MODALITY，空则匹配所有来源
- 转发目标节点：从已配置的节点中选择

### 转发队列
- 失败任务5分钟后自动重试，最多重试3次
- 点"▶ 重试失败"可手动立即重试
- 点"🗑 清除已完成"清理成功记录

---

## 📋 Worklist 页

### 启动 Worklist SCP（让设备来查询）
1. 填写 AE Title 和端口（默认 11113）
2. 点"▶ 启动服务"
3. 设备配置指向本机 IP 和端口即可查询

### 管理 Worklist 数据
- "➕ 添加项目"：手动填写患者ID和姓名，其他信息自动生成
- "🎲 生成测试数据"：生成10条随机数据用于测试
- 数据保存在 config/worklist_data.json，重启后保留

### SCU 查询（向其他服务器查询）
1. 填写服务器 AE、Host、Port
2. 可选填患者ID、姓名、模态作为过滤条件
3. 检查日期支持范围格式：
   - 20260203 — 精确日期
   - 20260101-20260131 — 日期范围
   - 20260101- — 某日期之后
   - -20260131 — 某日期之前
4. 点"🔍 查询"

## ✏️ 编辑器页

### 打开文件
点"📂 打开"选择 DCM 文件，左侧显示图像，右侧显示所有标签。

### 图像窗宽窗位
- 首次打开自动调整
- 拖动"窗位"和"窗宽"滑块手动调整
- 快速预设：肺窗（-600/1500）、纵隔（40/400）、骨窗（400/1800）、软组织（50/350）

### 匿名化
点"🔒 匿名化"，保留病历号后4位，其余患者信息清空。

### 修改 UID
点"🔑 修改UID"，重新生成所有 UID 和 AccessionNumber，PatientID 追加日期后缀。
修改后上传 PACS 不会报 409 冲突。

### 编辑标签
直接在右侧文本框中修改标签值，然后点"应用标签修改"。
支持字符串、整数（US/SS/UL/SL）、浮点（FL/FD）类型的标签。

### 计算年龄
仅当文件中无年龄字段时显示此按钮，根据出生日期自动计算并填入。

---

## 📁 文件浏览页

### 扫描文件夹
点"📂 选择文件夹"，自动扫描所有 .dcm 文件并列表显示。点击列标题可排序。

### 批量操作
所有批量操作输出到源文件夹同级的 _updated 目录，原文件不变。
操作过程中可点"✕ 取消"中断。

| 操作 | 说明 |
|------|------|
| 批量匿名化 | 保留病历号后4位 |
| 批量计算年龄 | 仅处理无年龄字段的文件 |
| 批量修改UID（保持Study关联） | 同一检查的多张图共享新 StudyUID |
| 批量修改UID（每文件独立Study） | 每个文件独立生成新 StudyUID，适合多患者混合文件夹 |

注意：如果之前处理过文件导致所有文件 StudyUID 相同，请用原始文件重新处理，选择"每文件独立Study"模式。

### 导出 Excel
点"📊 导出Excel"，导出当前列表到 .xlsx 文件，包含路径、患者信息和三个 UID 字段。

## 常见问题

**Q: 上传 PACS 报 409 Conflict**
A: 说明 SOPInstanceUID 已存在。用"批量修改UID"处理后再上传。如果仍然报 409，改用"每文件独立Study"模式。

**Q: 患者姓名显示乱码**
A: 程序会自动检测并修复 GB2312/GBK/GB18030 编码，通常无需手动处理。

**Q: Worklist 查询无结果**
A: 检查服务器 AE Title、Host、Port 是否正确，以及 Worklist SCP 服务是否已启动。

**Q: 接收到的文件在哪里**
A: 默认保存在 ./storage 目录，可在接收页修改存储路径。

**Q: 配置丢失了**
A: 配置保存在 config/app_config.json，删除该文件后重启会重新生成默认配置。
